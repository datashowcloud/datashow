Uma empresa tem um servidor de aplicações em execução em uma instância Amazon EC2. O servidor de aplicações precisa acessar o conteúdo em um bucket privado do Amazon S3.
Qual é a abordagem recomendada para atender a esse requisito?

<ok>
Crie uma IAM role com as permissões apropriadas. Associe a role à instância EC2.
Correto. As IAM roles são credenciais temporárias que expiram. As IAM roles são mais seguras do que as chaves de acesso de longo prazo porque reduzem o risco caso as credenciais sejam expostas acidentalmente.
Usar uma IAM role para conceder permissões a aplicações executadas em instâncias Amazon EC2. https://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles_use_switch-role-ec2.html

Configure uma conexão de peering de VPC para permitir a comunicação privada entre a instância EC2 e o bucket do S3.
Incorreto. Uma conexão de peering de VPC é uma conexão de rede entre duas VPCs que permite rotear o tráfego entre elas usando endereços IPv4 ou IPv6 privados. As conexões de peering de VPC não podem conectar uma VPC e um bucket S3.
O que é peering de VPC? https://docs.aws.amazon.com/vpc/latest/peering/what-is-vpc-peering.html

Crie uma chave de acesso compartilhada. Configure a instância EC2 para usar a chave codificada.
Incorreto. A criação de uma chave compartilhada não é uma prática recomendada porque isso reduz o valor da auditoria do acesso a recursos da AWS. Incorporar chaves de acesso ao código da aplicação não é uma prática recomendada porque o código pode ficar comprometido.
Melhores práticas para o gerenciamento de chaves de acesso da AWS. https://docs.aws.amazon.com/general/latest/gr/aws-access-keys-best-practices.html

Configure a aplicação para ler uma chave de acesso de uma fonte segura.
Incorreto. O uso de IAM roles em vez de chaves de acesso de longo prazo é uma prática recomendada. As chaves de acesso de longo prazo, como aquelas associadas a usuários do IAM e a usuários raiz da conta da AWS, permanecem válidas até que sejam revogadas manualmente. No entanto, as credenciais de segurança temporárias obtidas por meio de IAM roles e outros recursos do AWS Security Token Service (AWS STS) expiram após um curto período. Use credenciais de segurança temporárias para ajudar a reduzir o risco de que as credenciais sejam expostas acidentalmente.
Melhores práticas para gerenciar chaves de acesso da AWS. https://docs.aws.amazon.com/general/latest/gr/aws-access-keys-best-practices.html

Quais recursos ou produtos da AWS são compatíveis com a replicação de dados em todas as regiões AWS?
(Selecione DUAS respostas)

<ok>
Amazon S3
Correto. O Amazon S3 é compatível com a replicação entre regiões. Com a replicação entre regiões, você designa um bucket do S3 de destino em outra região. Quando a replicação entre regiões estiver ativada, qualquer novo objeto carregado será replicado no bucket do S3 de destino.
Replicar objetos. https://docs.aws.amazon.com/AmazonS3/latest/dev/crr.html

Amazon Elastic Block Store (Amazon EBS)
Incorreto. O Amazon EBS replica dados automaticamente em uma zona de disponibilidade. O Amazon EBS não é compatível com a replicação entre regiões.
Amazon Elastic Block Store. https://aws.amazon.com/ebs/

Amazon EC2 instance store
Incorreto. Um armazenamento de instância EC2 é o armazenamento em bloco anexado a uma instância EC2. Esse armazenamento está localizado em discos que estão anexados fisicamente ao computador host. Um armazenamento de instância é ideal para armazenamento temporário de informações que mudam com frequência. Os dados armazenados em um armazenamento de instância são temporários. Não há mecanismo integrado para replicar dados entre regiões.
Armazenamento de instâncias Amazon EC2. https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/InstanceStorage.html

AWS Storage Gateway
Incorreto. O Storage Gateway conecta um dispositivo de software local ao armazenamento baseado na nuvem. O Storage Gateway fornece integração com recursos de segurança de dados entre o ambiente de TI local e a infraestrutura de armazenamento da AWS, como o Amazon S3. O Storage Gateway não é compatível diretamente com a replicação entre regiões.
O que é o AWS Storage Gateway? https://docs.aws.amazon.com/storagegateway/latest/userguide/WhatIsStorageGateway.html

<ok>
Amazon RDS
Correto. Você pode usar o Amazon RDS para hospedar bancos de dados relacionais na AWS. Uma instância de banco de dados do RDS reside em uma única região. Com o Amazon RDS, você pode criar réplicas de leitura entre regiões. O Amazon RDS replica todos os dados da instância do banco de dados primário na réplica de leitura entre regiões.
Trabalhar com réplicas de leitura. https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/USER_ReadRepl.html

Amazon DynamoDB Accelerator (DAX)
Incorreto. O DAX é usado para reduzir o tempo de resposta de uma tabela do DynamoDB de um dígito em milissegundos para microssegundos. As tabelas do DynamoDB não podem hospedar sites estáticos.
Aceleração na memória com o DynamoDB Accelerator (DAX). https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/DAX.html

Amazon Route S3
Incorreto. O Route 53 é um serviço da Web de DNS altamente disponível e escalável. As três principais funções do Route 53 são registrar nomes de domínio, rotear o tráfego da Internet para os recursos do domínio e conferir a integridade desses recursos. O Route 53 pode direcionar o tráfego para buckets do S3. Mas como a pergunta descreve apenas um bucket do S3, o Route 53 teria apenas uma rota possível e não poderia reduzir a latência.
O que é o Amazon Route 53? https://docs.aws.amazon.com/Route53/latest/DeveloperGuide/Welcome.html
Roteamento de tráfego para um site hospedado em um bucket do Amazon S3. https://docs.aws.amazon.com/Route53/latest/DeveloperGuide/RoutingToS3Bucket.html

Amazon CloudFront
Correto. O CloudFront é um serviço da Web que acelera a distribuição do conteúdo estático e dinâmico da Web, como arquivos .html, .css, .js e de imagem aos usuários. O conteúdo é armazenado em cache nos pontos de presença. O conteúdo acessado repetidamente pode ser fornecido pelos pontos de presença em vez do bucket do S3 de origem.
Acelerar a entrega de conteúdo do site estático. https://docs.aws.amazon.com/AmazonCloudFront/latest/DeveloperGuide/IntroductionUseCases.html#IntroductionUseCasesStaticWebsite

Qual produto da AWS fornece uma solução de armazenamento de arquivos compartilhado, simples e escalável para uso com servidores locais e instâncias Amazon EC2 baseadas em Linux?

AWS Managed Services (AWS)
Incorreto. O AMS ajuda a operar sua infraestrutura da AWS de forma mais eficiente e segura. Usando os produtos da AWS e uma biblioteca crescente de automações, configurações e registros, o AMS pode aumentar e otimizar os recursos operacionais em ambientes da AWS novos e existentes. No entanto, o AMS não é um serviço de armazenamento.
AWS Managed Services. https://aws.amazon.com/managed-services/

Amazon S3 Glacier
Incorreto. O S3 Glacier é um object store usado para fins de arquivamento. Ele não é adequado para acesso único.
O que é o Amazon S3 Glacier? https://docs.aws.amazon.com/amazonglacier/latest/dev/introduction.html

Amazon Elastic Block Store (Amazon EBS)
Incorreto. O Amazon EBS fornece volumes de armazenamento em nível de bloco para uso com instâncias EC2. Os volumes do EBS se comportam como dispositivos de blocos brutos e não formatados. No entanto, os volumes do EBS não podem ser compartilhados entre servidores locais e instâncias EC2.
Amazon Elastic Block Store (Amazon EBS). https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/AmazonEBS.html

<ok>
Amazon Elastic File System (Amazon EFS)
Correto. O Amazon EFS fornece um sistema de arquivos elástico que permite compartilhar dados de arquivos sem a necessidade de provisionar e gerenciar o armazenamento. Ele pode ser usado com os serviços de nuvem da AWS e recursos locais, e foi criado visando dimensionar sob demanda para petabytes sem interromper as aplicações. Com o Amazon EFS, é possível aumentar e reduzir seus sistemas de arquivos automaticamente à medida que adiciona e remove arquivos, eliminando a necessidade de provisionar e gerenciar a capacidade para acomodar o crescimento.
Passo a passo: Crie e monte um sistema de arquivos no local com o AWS Direct Connect e a VPN. https://docs.aws.amazon.com/efs/latest/ug/efs-onpremises.html

Uma empresa precisa de acesso ininterrupto por telefone, e-mail e chat. O tempo de resposta deverá ser inferior a uma hora se um sistema de produção tiver uma interrupção de serviço.
Qual plano do AWS Support atende a esses requisitos pelo MENOR custo?

AWS Basic Support
Incorreto. O plano Basic Support não inclui acesso por chat nem chamadas telefônicas.
Comparar planos do AWS Support. https://aws.amazon.com/premiumsupport/plans/

AWS Developer Support
Incorreto. O plano Developer Support não inclui acesso por chat nem chamadas telefônicas. Ele fornece suporte por e-mail somente durante o horário comercial.
Comparar planos do AWS Support. https://aws.amazon.com/premiumsupport/plans/

<ok>
AWS Business Support
Correto. O plano Business Support fornece acesso ininterrupto por telefone, e-mail e chat. O plano Business Support tem um tempo de resposta inferior a um hora caso um sistema de produção sofra uma interrupção de serviço.
Comparar planos do AWS Support. https://aws.amazon.com/premiumsupport/plans/

AWS Enterprise Support
Incorreto. O plano Enterprise Support fornece acesso ininterrupto por telefone, e-mail e chat. O plano Enterprise Support tem um tempo de resposta inferior a 15 minutos caso um sistema de produção sofra uma interrupção de serviço. No entanto, o plano Enterprise Support é mais caro do que o Business Support.
Comparar planos do AWS Support. https://aws.amazon.com/premiumsupport/plans/

Qual modelo de preço do Amazon EC2 se ajusta com base na oferta e na demanda de instâncias EC2?

Instâncias On-Demand
Incorreto. As instâncias On-Demand são oferecidas a um preço definido de acordo com a região da AWS.
Preço on-demand do Amazon EC2. https://aws.amazon.com/ec2/pricing/on-demand/

Instâncias reservadas
Incorreto. As instâncias reservadas reservam capacidade a uma taxa com desconto. O cliente se compromete a comprar uma certa quantidade de computação.
Instâncias reservadas do Amazon EC2. https://aws.amazon.com/ec2/pricing/reserved-instances/

<ok>
Instâncias Spot
Correto. As instâncias spot são descontadas mais intensamente quando há mais capacidade disponível nas zonas de disponibilidade.
Instâncias spot do Amazon EC2. https://aws.amazon.com/ec2/spot/

Instâncias reservadas conversíveis
Incorreto. As instâncias reservadas reservam capacidade a uma taxa com desconto. O cliente se compromete a comprar uma certa quantidade de computação. Com as instâncias reservadas conversíveis, é possível alterar a família da instância, o sistema operacional e as locações.
Instâncias reservadas do Amazon EC2. https://aws.amazon.com/ec2/pricing/reserved-instances/

Qual das opções a seguir é uma vantagem do faturamento consolidado na AWS?

<ok>
Qualificação de preços por volume
Correto. O faturamento consolidado é um recurso do AWS Organizations. É possível combinar o uso em todas as contas na organização para compartilhar os descontos de preços por volume, os descontos de instância reservada e os Savings Plans. Essa solução pode resultar em uma carga menor em comparação com o uso de contas individuais independentes.
Faturamento consolidado para o AWS Organizations. https://docs.aws.amazon.com/awsaccountbilling/latest/aboutv2/consolidated-billing.html

Permissões de acesso compartilhado
Incorreto. As permissões de acesso compartilhado são um recurso de roles desenvolvido no AWS Identity and Access Management (IAM). Essa solução não está relacionada ao faturamento consolidado.
O que é o IAM? https://docs.aws.amazon.com/IAM/latest/UserGuide/introduction.html

Várias faturas para cada conta
Incorreto. O objetivo do faturamento consolidado é ter uma fatura para várias contas.
Faturamento consolidado para o AWS Organizations. https://docs.aws.amazon.com/awsaccountbilling/latest/aboutv2/consolidated-billing.html

Eliminação da necessidade de marcar recursos
Incorreto. No faturamento consolidado, é possível aplicar tags que representam categorias de negócios. Essa funcionalidade ajuda a organizar seus custos em vários serviços dentro do faturamento consolidado.
Usar tags de alocação de custos. https://docs.aws.amazon.com/awsaccountbilling/latest/aboutv2/cost-alloc-tags.html

Qual produto da AWS pode criar um alarme que envia uma notificação quando um limite de faturamento é excedido?

AWS Trusted Advisor
Incorreto. O Trusted Advisor baseia-se nas melhores práticas aprendidas ao atender centenas de milhares de clientes da AWS. O Trusted Advisor inspeciona seu ambiente da AWS e faz recomendações quando há oportunidades para economizar dinheiro, melhorar a performance do sistema ou ajudar a corrigir falhas de segurança. No entanto, o Trusted Advisor não pode criar um alarme de faturamento.
AWS Trusted Advisor. https://docs.aws.amazon.com/awssupport/latest/user/trusted-advisor.html

AWS CloudTrail
Incorreto. O CloudTrail é um produto da AWS que torna possível a governança, a compliance e a auditoria operacional e de risco na sua conta da AWS. O CloudTrail registra ações realizadas por um usuário, uma função ou um produto da AWS como eventos. No entanto, o CloudTrail não pode criar um alarme de faturamento.
O que é o AWS CloudTrail? https://docs.aws.amazon.com/awscloudtrail/latest/userguide/cloudtrail-user-guide.html

<ok>
Amazon CloudWatch
Correto. Você pode monitorar suas cobranças estimadas da AWS usando o CloudWatch. Quando você ativa o monitoramento de cobranças estimadas em sua conta da AWS, elas são calculadas e enviadas várias vezes ao dia para o CloudWatch como dados de métrica.
Criação de um alarme de faturamento para monitorar as cobranças estimadas da AWS. https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/monitor_estimated_charges_with_cloudwatch.html

Amazon QuickSight
Incorreto. O QuickSight é um serviço de business intelligence (BI) em escala de nuvem que você pode usar para fornecer insights fáceis de entender. O QuickSight se conecta aos seus dados na nuvem e combina dados de muitas fontes diferentes. No entanto, o QuickSight é uma ferramenta de visualização de dados, não uma ferramenta para criar alarmes.
O que é o Amazon QuickSight? https://docs.aws.amazon.com/quicksight/latest/user/welcome.html
